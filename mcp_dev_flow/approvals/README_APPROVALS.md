# Aprovações
Crie FASE_XX.approved com 'APROVADO'.

## Descrição
As aprovações são obrigatórias para avançar entre fases no MCP Dev Flow. Elas garantem que cada etapa seja validada antes de prosseguir, promovendo governança e reduzindo riscos.

## Como Aprovar
- Crie um arquivo `faseXX.approved` (ex.: `fase1.approved`) no diretório `approvals/`.
- Conteúdo: "APROVADO" ou detalhes da aprovação (ex.: "Aprovado por [Nome] em [Data]").
- Alternativa: Use comando "APROVADO FASE X" no chat do Copilot Agent.

## Exemplos
- fase1.approved: "APROVADO - Requisitos coletados."
- fase3.approved: "APROVADO - Arquitetura definida com DDD."
- fase5.approved: "APROVADO - Testes executados com 85% cobertura."
- fase8.approved: "APROVADO - Decisão Go: Prosseguir com ajustes."

Crie FASE_XX.approved com 'APROVADO'.

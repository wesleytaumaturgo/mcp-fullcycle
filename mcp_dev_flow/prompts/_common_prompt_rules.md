# Regras comuns de prompt

## Descrição
Regras gerais para todos os prompts no MCP Dev Flow, garantindo consistência e qualidade.

## Regras
- Sempre incluir contexto do projeto.
- Especificar saídas esperadas claramente.
- Usar critérios de aceitação mensuráveis.
- Integrar com componentes do MCP (Mesa, Codex, Super Auditor).
- Manter linguagem clara e objetiva.
- Evitar ambiguidades para garantir respostas precisas da IA.
